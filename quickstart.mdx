title: "wx-cli quickstart: install, init, and first queries"

---

This quickstart walks you through installing wx-cli, running `wx init` to extract your WeChat encryption keys, and issuing your first queries against local chat history. The entire setup takes under five minutes on a machine where WeChat is already installed and logged in.

<Steps>
  <Step title="Install wx-cli">
    The fastest path is npm, which works on all platforms:

    ```bash
    npm install -g @jackwener/wx-cli
    ```

    For curl, PowerShell, manual download, and build-from-source options, see the [installation guide](/installation).
  </Step>

  <Step title="macOS only: sign WeChat">
    On macOS, WeChat's process memory is protected. You must apply an ad-hoc code signature before wx-cli can scan it for encryption keys. Skip this step on Linux or Windows.

    <Tabs>
      <Tab title="macOS">
        ```bash
        # Sign WeChat (once per WeChat version)
        codesign --force --deep --sign - /Applications/WeChat.app

        # Restart WeChat and wait for it to fully log in
        killall WeChat && open /Applications/WeChat.app
        ```

        If `codesign` reports `signature in use`, remove the conflicting signature first:

        ```bash
        codesign --remove-signature \
          "/Applications/WeChat.app/Contents/Frameworks/vlc_plugins/librtp_mpeg4_plugin.dylib"
        codesign --force --deep --sign - /Applications/WeChat.app
        ```
      </Tab>
      <Tab title="Linux">
        No signing step required. Make sure WeChat is running and you are logged in, then proceed to the next step.
      </Tab>
      <Tab title="Windows">
        No signing step required. Make sure WeChat is running and you are logged in, then proceed to the next step. Run all `wx` commands in a PowerShell window opened **as Administrator**.
      </Tab>
    </Tabs>
  </Step>

  <Step title="Run wx init">
    With WeChat open and fully logged in, run the init command. On macOS and Linux this requires `sudo` so the tool can attach to WeChat's process:

    <Tabs>
      <Tab title="macOS">
        ```bash
        sudo wx init
        ```
      </Tab>
      <Tab title="Linux">
        ```bash
        sudo wx init
        ```
      </Tab>
      <Tab title="Windows">
        ```powershell
        wx init
        ```

        Run this in a PowerShell window opened as Administrator.
      </Tab>
    </Tabs>

    `wx init` does three things:

    1. **Detects the WeChat database directory** — automatically locates WeChat's `db_storage` folder under your home directory.
    2. **Scans process memory for encryption keys** — reads the running WeChat process to extract the SQLCipher raw keys held in RAM.
    3. **Writes configuration** — saves `config.json` and `all_keys.json` to `~/.wx-cli/` with permissions `0600` so only your user can read them.

    Successful output looks like:

    ```
    检测微信数据目录...
    找到数据目录: /Users/you/Library/Containers/com.tencent.xinWeChat/.../db_storage
    扫描加密密钥（需要 root 权限）...
    成功提取 12 个数据库密钥
    密钥已保存: /Users/you/.wx-cli/all_keys.json
    配置已保存: /Users/you/.wx-cli/config.json
    初始化完成，可以使用 wx sessions / wx history 等命令了
    ```

    If you need to re-run init (e.g., after a WeChat update changes the keys), add `--force`:

    ```bash
    sudo wx init --force
    ```

    <Note>
      You only need to run `wx init` once per WeChat installation. The daemon caches the decrypted databases and reuses them as long as the source files haven't changed on disk.
    </Note>
  </Step>

  <Step title="Verify: list recent sessions">
    Run `wx sessions` to confirm everything is working. The daemon starts automatically in the background on the first call.

    ```bash
    wx sessions
    ```

    Example output (YAML, the default):

    ```yaml
    - chat_name: "张三"
      chat_type: private
      last_message: "好的，明天见"
      unread_count: 0
      timestamp: "2026-04-18T21:30:00"
    - chat_name: "AI 交流群"
      chat_type: group
      last_message: "大家好"
      unread_count: 3
      timestamp: "2026-04-18T20:15:00"
    ```

    If you see a list of conversations, wx-cli is fully operational.
  </Step>

  <Step title="Explore: history, search, and more">
    A few commands to get you started:

    ```bash
    # Last 50 messages with a contact (fuzzy name match)
    wx history "张三"

    # Pull up to 2000 messages
    wx history "张三" -n 2000

    # Filter by date range
    wx history "AI群" --since 2026-04-01 --until 2026-04-15

    # Full-text search across all chats
    wx search "关键词"

    # Search within a specific chat with a date filter
    wx search "会议" --in "工作群" --since 2026-01-01

    # Sessions with unread messages (real contacts only)
    wx unread --filter private,group

    # Incremental new messages since last check
    wx new-messages
    ```
  </Step>
</Steps>

## Output format

wx-cli defaults to YAML output, which is compact and easy to read in a terminal or paste into an AI prompt. Use `--json` on any command to switch to JSON, which works well with `jq`:

```bash
# YAML (default)
wx sessions

# JSON
wx sessions --json

# JSON piped to jq
wx search "关键词" --json | jq '.[0].content'
```

Every session and message object includes a `chat_type` field with one of these values:

| Value | Meaning |
|---|---|
| `private` | One-on-one conversation |
| `group` | Group chat |
| `official_account` | Public account, subscription account, or system notification |
| `folded` | Folded subscriptions or collapsed group inbox |
